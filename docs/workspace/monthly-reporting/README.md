## 当前状况

已识别为跨模块报表能力需求（涉及 `reporting`、`customer`、`workshop-material`、`project`、`scheduler` 等模块），并已写入项目级需求。当前 5 项核心口径已由用户确认，后续可以进入详细设计与实施规划阶段。

## 已确认口径

1. **指标范围**: 按“全包含”处理，覆盖数量、金额、成本、结存、报废损耗、退料回补等核心指标。
2. **销售域口径**: 按“全包含”处理，除客户收发数量外，还包含销售金额、退货金额、毛利、回款等经营统计指标。
3. **生成方式**: 采用“每月固定正式月报 + 人工触发重算”，并允许按选择的日期范围生成报表。
4. **交付形式**: 系统内查看 + Excel 导出。
5. **补录处理**: 允许月末后补录单据并重算，但后续设计必须保证正式月报、重算结果与日期范围报表的追溯与区分。

## 下一步关注点

1. 明确“正式月报”“人工重算结果”“日期范围报表”三类结果的语义边界、版本标识与追溯策略。
2. 拆解公司整体、车间、销售域、研发项目四类月报的汇总/明细字段口径与数据来源。
3. 明确系统查看页面与 Excel 导出模板的首版范围，避免一期范围失控。
