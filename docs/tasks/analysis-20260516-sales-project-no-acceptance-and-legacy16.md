# 销售项目未验收行与库存选择问题分析

## 结论

`2026-05-16` 重新按旧库库存证据复核后，不能继续只用 `acceptance_date` 判断项目物料来源。

更可靠的拆分依据是：

- `saifute_product_material.material_id IS NOT NULL`
- `saifute_inventory_used.after_order_type = 8`
- `saifute_inventory_log.related_order_type = 8`

这三类证据共同指向“项目直接从已有库存选择 / 消耗”。`acceptance_date` 只能说明旧项目物料行上记录了验收日期，不能单独证明该行一定是到货验收入库。

当前有效项目 `756` 行的来源证据应重新拆成：

| 类别 | 判断依据 | 行数 | 处理方式 |
| --- | --- | ---: | --- |
| 到货验收 | `acceptance_date IS NOT NULL` 且无库存选择证据 | `380` | 才能走项目验收 / 入库 |
| 已有库存选择 | `material_id IS NOT NULL` 且有项目库存使用证据 | `122` | 不能伪造成验收入库，应走库存归属 / 选择模型 |
| 历史项目专用补录入库 | `acceptance_date IS NULL` 且 `material_id IS NULL` 且无项目库存使用证据，业务确认这些物料只用于对应项目 | `254` | 补建项目绑定验收单 / 入库单；验收日期取旧项目数据 `create_time` 的日期 |
| 作废项目暂不自动处理 | `del_flag = 2` | `2` | 待人工确认 |

所以，原先的 `501` 条 accepted-only 项目验收入库口径过宽：其中 `121` 行其实有已有库存选择证据，不应继续按到货验收单处理。

`2026-05-16` 进一步确认：这 `254` 行不是普通库存待选择。旧系统因为这些物料只用一次、只用于这些项目，没有创建验收单，也没有创建旧物料主数据，而是直接归属到项目。新系统需要补齐单据链和物料链。验收日期不能空；旧 `saifute_product_material` 明细表没有创建时间字段，补录验收日期取旧项目头 `saifute_composite_product.create_time` 的日期。

针对用户追问的“没有验收日期的数据”：

- 无验收日期总数仍是 `255`。
- 其中 `254` 行没有库存选择证据，也不是普通库存待选择；业务上应作为历史项目专用补录入库。
- 只有 `1` 行能确认是从已有库存选择：legacy 行 `713`，项目 `18` 凤凰山硐室，物料 `压缩氧自救器 / ZY45X`，数量 `60`，来源为退货单 `TH20260122002` 明细 `33`，旧库存日志在 `2026-05-06 09:53:19` 对项目扣减 `60`。

### 处理说明

无验收日期行不能一律按待选择处理。后续处理要继续区分：

- 有库存选择证据的 `1` 行：不能伪造成验收入库，应进入已有库存选择 / 项目归属调整类模型。
- 没有库存选择证据且旧物料主数据为空的 `254` 行：按历史项目专用补录入库处理，验收日期写旧项目 `create_time` 的日期。

正确规则：

- `380` 行保留旧 `acceptance_date` 作为验收日期。
- `254` 行使用旧项目 `create_time` 的日期作为补录验收日期。
- `122` 行已有库存选择证据仍不得伪造成验收单。
- 补录入库必须通过项目绑定验收单 / 入库单生成项目库存和价格层，不直接手写库存流水。

## 后续执行建议

### 第一步：刷新审计脚本分类

分类规则应从 `acceptance_date` 单一字段改为：

- 到货验收：`acceptance_date IS NOT NULL` 且无库存选择证据。
- 已有库存选择：`material_id IS NOT NULL` 且存在 `saifute_inventory_used.after_order_type = 8` 或 `saifute_inventory_log.related_order_type = 8`。
- 历史项目专用补录入库：无验收日期、无库存选择证据、旧物料主数据为空且业务确认只用于项目；验收日期取旧项目 `create_time`。

### 第二步：修正 accepted-only 项目验收入库范围

当前 `501` 行 accepted-only 口径包含 `121` 行已有库存选择证据。确认“已有库存选择不走验收单”后，项目入库目标应刷新为 `380 + 254 = 634` 行，其中 `254` 行使用项目创建日期作为补录验收日期。

### 第三步：项目库存选择 / 出库

已有库存选择 / 归属调整单独处理：

- `122` 行已有库存选择证据不得进入补录验收单。
- 后续需要独立恢复 / 表达普通库存转项目或已消耗事实。
