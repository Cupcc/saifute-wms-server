# 项目 / 研发主题需求

## Metadata

- ID: `domain-project-management`
- Status: `needs-confirmation`
- Scope: `domain-level`

## 主题定义

- `project` 是项目 / 研发归集主题，解决“项目是什么、项目消耗了什么、项目还剩多少可归集余额、项目成果成本如何查看”的问题。
- 本主题关注项目台账、项目物料消耗、项目成本归集、轻量 `BOM` 参考和项目成果归集；它不是新的物理仓库，也不替代 `RD 小仓` 的仓别协同职责。
- 第一阶段项目范围以内部研发项目为主，不以正式销售、收入确认或对外交付结算为核心。

## 承接的项目级能力版图

### 项目 / 研发（`project`）

- 本 domain 承接项目总纲中 `project` 模块的长期职责，作为项目台账、项目归集与项目成本口径的真源。
- `rd-subwarehouse` 负责“主仓与 RD 小仓如何协同、物料如何过仓、RD 采购状态如何流转”；本 domain 负责“项目如何建档、如何归集消耗、如何看余额与成本”。
- 项目页面需要作为独立入口存在，供仓库管理员维护项目台账，并在项目上下文中录入或查看相关业务单据。
- 项目相关的真实库存动作至少包括：`项目采购入库`、`项目领料`、`项目退料`、`项目报废`；这些动作都必须继续通过共享库存能力落账，而不是在项目表里直接维护库存。
- 第一阶段保留轻量 `BOM` / 标准用料清单作为参考能力，用于辅助项目成果成本归集，但不引入复杂版本、替代料或强控发料逻辑。

## 长期约束

- `C1` 项目不等于物理库存池：项目是归集与核算维度，不是新的真实仓别；真实库存范围仍以 `MAIN` / `RD_SUB` 为准。状态：`生效中`
- `C2` 库存写入口统一：项目相关的任何领料、退料、报废、采购回补都必须通过 `inventory-core` 产生库存事实，不能由项目模块旁路改库存。状态：`生效中`
- `C3` 项目归集与仓别分离：项目归属、项目余额和项目成本可以独立展示，但不能把“项目归属库存”误当成新的库存唯一键。状态：`生效中`
- `C4` RD 项目强绑定：当项目消耗发生在 `RD 小仓` 侧时，单据必须绑定研发项目或项目式归集项，不能脱离项目独立流转。状态：`生效中`
- `C5` 轻审核起点：项目域第一阶段默认不接 `workflow`，`auditStatusSnapshot` 固定按 `NOT_REQUIRED` 处理；若后续需要审核，应作为独立增量能力确认。状态：`生效中`
- `C6` 轻量 BOM 原则：`BOM` 仅承担参考、预估和成果归集辅助作用，不承担强制发料、版本控制或替代料引擎语义。状态：`生效中`
- `C7` 项目成本优先于收入：第一阶段优先确保项目投入、领用、退料、报废、结余等成本口径清晰，不把项目域扩展为销售结算域。状态：`生效中`

## 长期业务口径

### 项目台账口径

- 项目台账由仓库管理员或受授权人员维护，至少需要支撑项目建档、启停、查询、导出和项目上下文录单。
- 当前阶段项目优先承接内部研发项目；如出现办公用品或通用耗材，也应挂到明确的项目式归集项，而不是脱离项目裸流转。
- 项目页面中的“项目库存”第一阶段不表示独立物理库存，而是表示项目归属余额、项目消耗台账余额或可归集剩余。

### 项目事务口径

- 当前项目事务目标覆盖：`项目采购入库`、`项目领料`、`项目退料`、`项目报废`。
- 与 `RD 小仓` 有关的项目领料、退料、报废默认作用于 `RD_SUB`；涉及采购到货时，仍先遵循主仓验收入 `MAIN`、再交接到 `RD_SUB` 的既有链路。
- 项目修改不能直接覆盖旧明细，必须按差量重算库存副作用、来源追溯和项目成本。
- 项目作废或删除若允许执行，必须保留历史事实，并通过逆操作恢复库存结果，而不是静默抹平历史。

### 项目成本与成果口径

- 项目成本至少需要按项目查看累计采购投入、项目领料消耗、项目退料回补、项目报废损耗，以及项目维度的汇总与明细。
- 项目成果、样品发出或阶段性交付记录，第一阶段主要用于回答“这次成果用了哪些料、成本是多少”，不直接替代客户出库单或销售结算单。
- 项目页面应能同时提供项目总表、项目物料明细、项目成本汇总和项目台账余额等视图。

## 阶段路线图

| 阶段 | 目标 | 当前状态 |
| --- | --- | --- |
| Phase 1 | 收敛项目台账、项目物料消耗和差量库存联动的基础闭环 | `进行中` |

## 待确认

- 项目主档是否需要显式区分 `立项 / 进行中 / 暂停 / 关闭` 等业务状态，还是先按轻量启停管理。
- 项目成果 / 样品发出是否只做成本归集记录，还是需要与 `customer` 域建立显式上下游引用。
- 非 `RD_SUB` 场景下，是否允许存在直接从主仓 `MAIN` 发生的项目领用链路。
